# MapTR: Data Collection

## Overview

MapTR is trained and evaluated on large-scale autonomous driving datasets that provide multi-camera imagery paired with vectorized HD map annotations. The primary datasets used are nuScenes and Argoverse 2, both offering dense sensor data captured in urban environments with corresponding high-definition map ground truth.

---

## nuScenes Dataset

### Dataset Overview

The nuScenes dataset is the primary benchmark for MapTR evaluation.

| Property | Value |
|----------|-------|
| Total scenes | 1000 |
| Training scenes | 700 |
| Validation scenes | 150 |
| Test scenes | 150 |
| Keyframes per scene | 40 (at 2 Hz) |
| Total keyframes | 40,000 |
| Capture locations | Boston, Singapore |
| Weather/lighting | Day, night, rain, varying conditions |

### Camera Configuration

nuScenes employs 6 surround-view cameras providing full 360-degree coverage:

| Camera | Field of View | Resolution |
|--------|--------------|------------|
| CAM_FRONT | 70° horizontal | 1600 x 900 |
| CAM_FRONT_LEFT | 70° horizontal | 1600 x 900 |
| CAM_FRONT_RIGHT | 70° horizontal | 1600 x 900 |
| CAM_BACK | 70° horizontal | 1600 x 900 |
| CAM_BACK_LEFT | 70° horizontal | 1600 x 900 |
| CAM_BACK_RIGHT | 70° horizontal | 1600 x 900 |

All cameras are synchronized and calibrated. Intrinsic parameters (focal length, principal point, distortion coefficients) and extrinsic parameters (rotation and translation relative to ego vehicle frame) are provided for each camera.

### Map Expansion Pack

The nuScenes map expansion pack provides vectorized map annotations essential for MapTR:

- **Format**: Vectorized polylines and polygons stored in JSON
- **Coverage**: Full map coverage for all capture locations (Boston Seaport, Singapore OneNorth, Singapore Queenstown, Singapore Holland Village)
- **Map layers available**:
  - Lane dividers (polylines)
  - Road boundaries (polylines)
  - Pedestrian crossings (polygons)
  - Road segments (polygons)
  - Walkways (polygons)
  - Stop lines (polylines)
  - Carpark areas (polygons)

For MapTR, the standard evaluation uses three categories:
1. **Pedestrian crossing** (closed polygon)
2. **Lane divider** (polyline)
3. **Road boundary** (polyline)

### Ground Truth Generation

The vectorized ground truth for each frame is generated by:

1. Loading the global map for the current location
2. Querying map elements within the perception range of the ego vehicle
3. Transforming global coordinates to the ego-vehicle coordinate frame
4. Clipping elements to the perception range boundaries
5. Sampling/interpolating each element to a fixed number of points (N_pts = 20)

## Data Format and Structure

### Input Data Per Frame

Each training/evaluation sample consists of:

```
sample = {
    "images": [img_0, img_1, ..., img_5],       # 6 camera images (H x W x 3)
    "intrinsics": [K_0, K_1, ..., K_5],          # Camera intrinsic matrices (3 x 3)
    "extrinsics": [T_0, T_1, ..., T_5],          # Camera-to-ego transforms (4 x 4)
    "ego2global": T_ego2global,                   # Ego-to-global transform (4 x 4)
    "timestamp": t,                               # Frame timestamp
    "map_elements": [elem_0, elem_1, ..., elem_M] # Vectorized GT map elements
}
```

### Camera Intrinsic Matrix

```
K = [[fx,  0, cx],
     [ 0, fy, cy],
     [ 0,  0,  1]]
```

Where:
- `fx, fy`: Focal lengths in pixels
- `cx, cy`: Principal point coordinates

### Camera Extrinsic Matrix

The 4x4 transformation matrix from camera frame to ego-vehicle frame:

```
T_cam2ego = [[R_00, R_01, R_02, t_x],
             [R_10, R_11, R_12, t_y],
             [R_20, R_21, R_22, t_z],
             [   0,    0,    0,   1]]
```

### Vectorized Map Element Format

Each ground truth map element is stored as:

```
map_element = {
    "category": int,              # Class label (0=ped_crossing, 1=divider, 2=boundary)
    "points": [[x_1, y_1],       # Ordered point sequence in ego-vehicle BEV
               [x_2, y_2],
               ...
               [x_N, y_N]],      # N = N_pts (typically 20)
    "type": str                   # "polyline" or "polygon"
}
```

## Perception Range

### Standard Configuration

The perception range defines the spatial extent around the ego vehicle where map elements are predicted:

| Axis | Range | Total Span |
|------|-------|-----------|
| X (longitudinal) | [-30.0m, +30.0m] | 60m |
| Y (lateral) | [-15.0m, +15.0m] | 30m |

This defines a 60m x 30m rectangular region in the BEV plane centered on the ego vehicle.

### Alternative Configurations

Some experiments use extended ranges:

| Configuration | X Range | Y Range |
|--------------|---------|---------|
| Standard | [-30, 30] | [-15, 15] |
| Extended | [-50, 50] | [-25, 25] |
| Long-range | [-60, 60] | [-30, 30] |

### BEV Grid Resolution

The BEV feature map discretizes the perception range:
- Typical BEV resolution: 200 x 100 grid cells
- Cell size: 0.3m x 0.3m (for standard 60m x 30m range)

## Map Element Categories

### Pedestrian Crossing

- **Geometry**: Closed polygon (first point = last point conceptually)
- **Typical shape**: Rectangular region spanning road width
- **Point count**: Sampled to N_pts = 20 points along polygon boundary
- **Semantic meaning**: Designated area for pedestrian road crossing

### Lane Divider

- **Geometry**: Open polyline
- **Typical shape**: Curves following lane center/edge markings
- **Point count**: Sampled to N_pts = 20 points along polyline
- **Semantic meaning**: Physical or painted lane separation (white/yellow lines, curbs)

### Road Boundary

- **Geometry**: Open polyline
- **Typical shape**: Follows the outer edge of the drivable surface
- **Point count**: Sampled to N_pts = 20 points along polyline
- **Semantic meaning**: Boundary between road surface and non-road areas (sidewalk, vegetation, barriers)

## Data Augmentation

### Image-Level Augmentation

- Random horizontal flip (with corresponding map element reflection)
- Random resize (scale factor 0.5 to 1.5)
- Random crop to training resolution
- Color jitter (brightness, contrast, saturation)
- Normalization to ImageNet statistics

### BEV-Level Augmentation

- Random rotation around ego vehicle origin
- Random scaling of the BEV coordinate system
- Random translation (shift perception window)
- Note: All augmentations are applied consistently to both images (via modified extrinsics) and map GT

### Temporal Augmentation

- Random frame selection within keyframe neighborhoods
- For temporal models: random dropping of history frames

## Data Loading Pipeline

### Preprocessing Steps

1. **Image loading**: Read 6 camera images, resize to input resolution (typically 800 x 480 or 1600 x 900)
2. **Calibration loading**: Parse camera intrinsic and extrinsic matrices
3. **Map query**: Extract map elements within perception range from global map database
4. **Coordinate transform**: Convert map elements from global frame to ego-vehicle frame
5. **Element clipping**: Clip polylines/polygons to perception range boundaries
6. **Point sampling**: Resample each element to exactly N_pts points via linear interpolation
7. **Normalization**: Normalize point coordinates to [0, 1] range relative to perception range

### Data Loader Configuration

```python
data_config = {
    "input_size": (800, 480),          # Image resize resolution
    "num_cameras": 6,                   # Number of surround-view cameras
    "perception_range": [-30, 30, -15, 15],  # [x_min, x_max, y_min, y_max] in meters
    "num_points": 20,                   # Fixed points per map element
    "map_classes": ["ped_crossing", "divider", "boundary"],
    "max_elements": 100,                # Maximum GT elements per frame
    "augmentation": True,
    "normalize_coords": True            # Normalize to [0, 1]
}
```
